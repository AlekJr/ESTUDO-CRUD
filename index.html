Tutorial Blazor

List<T>: Representa uma lista fortemente tipada de objetos que podem ser acessados por índice. Fornece métodos para pesquisar, classificar e manipular listas.


For: No caso do for usamos para fazer um contador a cada click o contador vai adicionando um a mais.


Foreach: A instrução foreach executa uma instrução ou um bloco de instruções para cada elemento em uma instância. Um desses exemplo é o deletar item.


Lista não ordenada: representa uma lista de itens sem ordem rígida, isto é, uma coleção de itens que não trazem uma ordenação numérica e as suas posições, nessa lista, são irrelevantes.

exemplos:

  • Primeiro time
  • Segundo time
  • Terceiro time

Lista Ordenada: criamos tópicos de textos para uma melhor exemplificação de um determinado assunto. São recursos extremamente usados, inclusive quando nem imaginamos que ele esta sendo usado, como no caso de menus.

Exemplos:

  1. São Paulo
  2. Flamengo
  3. Coritiba
  4. Santos

Tabelas: faz parte do conjunto de elementos da linguagem de marcação e é utilizado para exibir dados de maneira tabular e estruturada.

exemplo:

Time Estado Saldo de gols
Santos SP 23
Flamengo RJ 32
Atlético Mineiro BH 27
São Bento SP 3

Listar Objetos: é usado uma List<categoria> que pega do repositorio e gera uma nova lista.


Deletar objetos: Usando a mesma List<categoria> que agora pega um ID do repositorio chamado delete e com isso ele deleta um objeto.


Cadastrar objetos: Criando uma new categoria no irepository<categoria> e depois criando uma task que adiciona no repositorio e com isso gera um novo cadastro.


Editar objetos: usando a task edit no irepository<categoria> e depois usando .update(categoria) no repositorio acaba possibilitando o usuario de editar os objetos já criados.try catch serve para que, em um dado momento em que um código possa gerar um erro inesperado, o programador consiga manipular as possibilidades e exceções. Dessa forma, através do Try ele irá tentar executar o código, caso não ocorra nenhum erro, o programa seguirá o seu fluxo normal.


Formulario: Criando um input para que o usuario possa prencher o Formulario e depois criando um botao para poder salvar as informações.


Injeção de indenpendencia: permite uma Injeção de indenpendencia do lado de fora de uma classe.
